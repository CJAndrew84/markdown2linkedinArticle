How to use this

1 · Write Markdown

  • Headings #–####, **bold**, *italic*, ~~strike~~.
  • Lists with -/1.; indent two spaces to nest.
  • > blockquotes, --- rules, `inline code`, fenced ``` blocks, [links](url).
  • Wrapped lines flow into one paragraph; end a line with two spaces (or \) to force a line break.

2 · Preview the result

  • The middle pane renders live as you type.
  • Switch LinkedIn Article / Newsletter in the top bar — the preview and HTML both follow the mode.

3 · Images

  • Markdown images ![alt](https://…) render directly. Add options: ![alt](url){width=400 align=center caption="My caption"}.
  • For local images: paste, drag-drop, or “Add image”. They’re embedded as base64 data URLs and listed in the Images panel where you set width, alignment, caption and alt.
  • Local images are referenced as ![alt](img:ID) in the editor — the panel controls the rest, so the editor stays readable.

4 · Mermaid diagrams

  • Use a fenced ```mermaid block (flowcharts, sequence, etc.). It renders to an image automatically.
  • The library loads on demand: a local mermaid.min.js next to this file (best for offline), else a CDN. Without either, the block stays as code.
  • LinkedIn and email strip SVG, so diagrams become PNG. Hover a diagram in the preview for Download PNG / Copy image.
  • For LinkedIn, the reliable route is Download PNG then upload via the Article editor’s image button — paste of a diagram is hit-and-miss.

5 · Paste into a LinkedIn Article

  1. Pick LinkedIn Article mode.
  2. Click Copy rendered.
  3. In LinkedIn’s Article editor, click into the body and paste (Ctrl/Cmd-V).
  4. LinkedIn keeps headings, bold/italic, lists, quotes, links and images, but strips most styling — that’s expected, and why this mode outputs clean semantic tags.

6 · Copy raw HTML for newsletters

  1. Pick Newsletter mode (inline styles, 600px email shell, mobile-friendly).
  2. Click Copy raw HTML and paste into your email tool’s HTML / code view.
  3. Or Copy rendered to paste formatted content into a WYSIWYG email composer.

Gotchas

  • Base64 images bloat the HTML. Fine for LinkedIn paste; for newsletters, host images and use real URLs where you can — some email tools reject data URLs.
  • Outlook desktop ignores max-width; Newsletter mode uses a fixed 600px table to cope.
  • Mermaid diagrams export as PNG data URLs — fine for LinkedIn upload, but heavy for email. For newsletters, download the PNG and host it, or use the Download/Copy buttons.
  • Copy rendered needs a clipboard-permitting context. Opening the file directly works in most browsers; if a copy ever fails, the tool falls back automatically.
